# IPA 輸入法 — P3 POC 安裝與測試指引

**版本:** 0.1.0(POC)
**檔案:** `ipa.schema.yaml`(方案)、`ipa.dict.yaml`(碼表,497 條)、`default.custom.yaml`(掛載)、`gen_rime.py`(產生器)
**前提:** Windows 已安裝小狼毫(Weasel)。

---

## 1. 安裝步驟

1. 開啟使用者資料夾:系統匣右鍵小狼毫圖示 →「用戶文件夾」(即 `%AppData%\Rime`)。
2. 將 `ipa.schema.yaml`、`ipa.dict.yaml` 複製進去。
3. `default.custom.yaml`:若該資料夾已有此檔,把本檔 `patch:` 下那兩行**合併**進既有檔案的 `patch:` 段;若沒有,直接複製整檔。
4. 系統匣右鍵 →「重新部署」。首次部署會編譯碼表,稍等數秒。
5. 在任一輸入框按 **F4**,選「IPA 國際音標」。

若部署後方案沒出現:開「輸入法設定」勾選方案,或檢查 `%TEMP%\rime.weasel.*` 日誌中是否有 YAML 錯誤。

## 2. POC 驗證點(依風險排序)

### V1 注音式緩衝與 Enter 上屏(最高風險)
本方案用 `fluid_editor`(語句流編輯器):選字後**留在組字緩衝**,不立即上屏。
- 打 `alp` → 選 t → 打 `asp` → 選 ʰ → 緩衝應顯示 tʰ → 按 **Enter**,整串 tʰ 上屏。
- **若 Enter 上屏的是原始編碼**(如 `alpasp`)而非轉換結果:記錄行為,改試把 schema 中 `fluid_editor` 換回 `express_editor`(逐字上屏模式)並記錄差異。這是 POC 的核心觀測。

### V2 關閉自動上屏 / 前綴共存
- 打 `s`:應顯示候選(s 及 sh、sbr… 等補全),**不得**自動上屏。
- 繼續打成 `sh`:候選出現 ʃ。確認 `s`→`sh` 的前綴延伸順暢。

### V3 標點優先序(0.1.1 已修,請複測)
`.`、`|`、`\` 已補「自指最高權重」候選,並自 punctuator 移除(交 speller)。
- 打 `.` → **第一候選應為 `.` 本身**(其次才是描述式 `sbr` 等);選字後上屏。
- 打 `\t.` → 尾端 `.` 應**立即**接上並顯示候選 ʈ(不再需多打一鍵)。〔V3-b 修正,重點複測〕
- 打 `|` → 第一候選 `|` 本身;`||` → ‖;`|13` → ˩˧(調値,見 V5)。
- 打 `\` → 第一候選 `\` 本身;續打成 `\sh` → ʃ。

### V4 四層編碼抽測
- ʃ:`S`(X-SAMPA)/ `sh`(助記)/ `paf`(描述式)/ `\sh`(Praat)。
- ɡ:`g` / `vvep` / `\gs`。ø:`2` / `cmfr` / `eu` / `\o/`。
- 分隔符(**分號**):打 `t;s` 應得 t、s 兩符號;打 `ts` 應見 t͡s 候選。

### V5 附加符號、聲調、上標(碼已改版)
- 附加符號:`alp` 選 t → `dnt` 選 ◌̪ → 緩衝 t̪ → Enter。
- **組合調符**:改用 X-SAMPA,如 é 打 `e` 選、再打 `_H`(高調符)。(原 `td` 已取消)
- **調値字母**:`|` + 數字(每位 1–5)。`|2`→˨、`|13`→˩˧、`|232`→˨˧˨。(原 `tl` 已取消)
- **上標數字**:`^` + 數字。`^35`→³⁵、`^214`→²¹⁴、`^70`→⁷⁰(2 位可 0–9,3 位限 1–5)。
- 塞擦組合式:`alp` `tie` `allf` → t͡ɬ。
- 上屏後在 Word/瀏覽器檢查疊加顯示(記事本弱,建議裝 Charis SIL)。
- 已知限制:分次上屏的「基底+組合符」無法跨次 NFC 合成;常用組合 P4 預生成解決。

### V6 中英切換
- Shift(或 F4 內開關)切 ABC 直打英文,再切回 IPA。

### V7 numpad 選字(0.1.1 新增,請複測)
- 叫出候選後,按 **numpad 1–9** 應選第 1–9 候選。
- 同時確認:**主鍵盤數字列**在組字中送**字面數字**(供 `|13`、`^35` 等碼),不誤選字。
- 〔風險〕若 numpad 按下反而把數字送進組字串(被 speller 吃掉),請記錄——
  此為「數字在 alphabet」的已知互動風險,回傳後改用 Space/方向鍵選字或改綁定。

## 3. 本 POC 的設計決定(供對照)

- `speller/auto_select: false`:注音式核心。
- `speller/delimiter: " ;"`:分隔符採 **分號**(全碼掃描確認未被任何碼使用)。
- 組合調符用 X-SAMPA `_T/_H/_M/_L/_B`;調値字母 `|`+數字;上標 `^`+數字。
- X-SAMPA 顎化採 `_j`(`'` 讓給 Praat 重音碼 `\'1`)。
- 標點 `. | \`:自指候選最高權重(999)+ 移出 punctuator,由 speller 取用。
- numpad 綁選字;主鍵盤數字列送字面數字。
- alphabet 由產生器自碼表字元集生成(87 字元),不手寫。
- 權重:X-SAMPA 500 > 助記 400 > 描述式 300 > Praat 200(標點自指 999)。
- 候選說明(顯示中文描述)屬 P5,規劃以 librime-lua filter 實作。

## 4. 測試結果記錄格式
每個驗證點記:通過 / 行為偏差(描述實際行為)/ 失敗。重點複測 **V3-b、V7**。
回傳後據以修正,出 0.1.2。

## 5. 重新產生
修改 `IPA_master.csv` 的源頭 `build_master.py` 後:
```
python3 build_master.py
python3 gen_rime.py --outdir . [--lang zh|en]
```
產生器內建全層撞名檢查與 delimiter 衝突檢查,任何新增碼若撞名會直接中止。
