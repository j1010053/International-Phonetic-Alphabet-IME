# IPA 輸入法 — P3 POC 安裝與測試指引

**版本:** 0.1.0(POC)
**檔案:** `ipa.schema.yaml`(方案)、`ipa.dict.yaml`(碼表,497 條)、`default.custom.yaml`(掛載)、`gen_rime.py`(產生器)
**前提:** Windows 已安裝小狼毫(Weasel)。

---

## 1. 安裝步驟

1. 開啟使用者資料夾:系統匣右鍵小狼毫圖示 →「用戶文件夾」(即 `%AppData%\Rime`)。
2. 將 `ipa.schema.yaml`、`ipa.dict.yaml` 複製進去。
3. `default.custom.yaml`:若該資料夾已有此檔,把本檔 `patch:` 下那兩行**合併**進既有檔案的 `patch:` 段;若沒有,直接複製整檔。
4. 系統匣右鍵 →「重新部署」。首次部署會編譯碼表,稍等數秒。
5. 在任一輸入框按 **F4**,選「IPA 國際音標」。

若部署後方案沒出現:開「輸入法設定」勾選方案,或檢查 `%TEMP%\rime.weasel.*` 日誌中是否有 YAML 錯誤。

## 2. POC 驗證點(依風險排序)

### V1 注音式緩衝與 Enter 上屏(最高風險)
本方案用 `fluid_editor`(語句流編輯器):選字後**留在組字緩衝**,不立即上屏。
- 打 `alp` → 選 t → 打 `asp` → 選 ʰ → 緩衝應顯示 tʰ → 按 **Enter**,整串 tʰ 上屏。
- **若 Enter 上屏的是原始編碼**(如 `alpasp`)而非轉換結果:記錄行為,改試把 schema 中 `fluid_editor` 換回 `express_editor`(逐字上屏模式)並記錄差異。這是 POC 的核心觀測。

### V2 關閉自動上屏 / 前綴共存
- 打 `s`:應顯示候選(s 及 sh、sbr… 等補全),**不得**自動上屏。
- 繼續打成 `sh`:候選出現 ʃ。確認 `s`→`sh` 的前綴延伸順暢。

### V3 標點優先序
`.`、`|`、`\` 等已納入 alphabet,組字時應視為「碼」:
- 打 `.` → 應進入組字並顯示候選(音節界 .),而非直接當標點上屏。
- 打 `\t.` → 尾端的 `.` 應接在碼上(候選 ʈ),不得中途斷開。
- 打 `|` → 候選為小句群 |;打 `||` → 大句群 ‖。
- 已知取捨:一般句號也會先進組字、需選字上屏。記錄使用感受,若不可接受則後續以 recognizer/punctuator 調整。

### V4 四層編碼抽測
同一符號四種打法皆應到達:
- ʃ:`S`(X-SAMPA)/ `sh`(助記)/ `paf`(描述式)/ `\sh`(Praat)。
- ɡ:`g` / `vvep` / `\gs`。ø:`2` / `cmfr` / `eu` / `\o/`。
- 分隔符:打 `t;s` 應得 t、s 兩符號(而非 t͡s);打 `ts` 應見 t͡s 候選。

### V5 附加符號與疊加
- `alp` 選 t → `dnt` 選 ◌̪ → 緩衝 t̪ → Enter。
- 聲調:母音後打 `td4`(組合調符)、獨立打 `tl15`(˩˥)。
- 塞擦組合式:`alp` `tie` `allf` → t͡ɬ。
- 在 Word/記事本檢查上屏後的疊加顯示(建議先裝 Charis SIL 或 Gentium)。
- 已知限制:分次上屏的「基底+組合符」無法跨次 NFC 合成(如 e+́ 不會合成 é);常用組合將於 P4 預生成整列解決。

### V6 中英切換
- Shift(或 F4 內開關)切到 ABC 直打英文,再切回 IPA。

## 3. 本 POC 的設計決定(供對照)

- `speller/auto_select: false`:注音式核心。
- `speller/delimiter: " ;"`:分隔符採 **分號**。原定 `'` 因 Praat 碼(`\'1`、`\''` 等)含單引號而衝突,經全碼掃描改用未被任何碼使用的 `;`(其餘可選:`# $ * [ ]`)。
- X-SAMPA 顎化採 `_j`(標準允許 `'` 或 `_j`;`'` 讓給 Praat 碼)。
- alphabet 由產生器**自碼表實際字元集生成**(87 字元),不手寫。
- 權重:X-SAMPA 500 > 助記 400 > 描述式 300 > Praat 200,影響補全候選排序。
- 候選說明(顯示中文描述)屬 P5,POC 不含;規劃以 librime-lua filter 實作。

## 4. 測試結果記錄格式
每個驗證點記:通過 / 行為偏差(描述實際行為)/ 失敗。V1 若偏差,附「express_editor 對照結果」。回傳後據以修 schema、出 0.1.1。

## 5. 重新產生
修改 `IPA_master.csv` 的源頭 `build_master.py` 後:
```
python3 build_master.py
python3 gen_rime.py --outdir . [--lang zh|en]
```
產生器內建全層撞名檢查與 delimiter 衝突檢查,任何新增碼若撞名會直接中止。
