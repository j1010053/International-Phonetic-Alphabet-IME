# Changelog

## 0.2.0 — P4 碼表完整化(全 2020 IPA 覆蓋 + 組合預生成)

### 真因記錄(V3-b, 使用者診斷)
- 元凶為 default key_binder 將 `.` 綁為翻頁鍵(when: has_menu, accept: period,
  send: Page_Down),組字中 `.` 被攔去翻頁、未進 speller,故候選不受影響。
- 連帶: default 亦綁 `,` 與 `-`/`=` 翻頁——三者皆為本方案碼字元,同樣會被吃。
- **產生器已固化修法: key_binder 不 import default,只留 numpad 綁定**,
  防止重產退回。翻頁改用 Page Up / Page Down。

### 完整性補齊(對照 2020 官方表)
- **補回組合調符**(0.1.2 取消 td 時誤刪整批): X-SAMPA `_T/_H/_M/_L/_B/_R/_F`。
- 新增 2020 連調欄擴充: ◌᷄ 高升 `hrise`、◌᷅ 低升 `lrise`、◌᷈ 升降 `rfall`。
- 新增 ɚ(`@``/`mdcrho`)、ɝ(`3``/`omcrho`)、ɫ(`5` 與 `l_e` 雙碼)。

### 組合預生成(combo, 268 列, NFC 輸出)
- 送氣 9、擠喉 11、鼻化母音 28、帶調母音 196(é ǎ 等直接出 NFC 預組形,
  解「分次上屏無法合成」限制)、成音節 5、清化響音 8、齒化 4、唇化 5、ɫ 2。
- 罕見組合仍走逐符號疊加,不受影響。

### 修正
- build_master.py 輸出路徑由寫死之容器路徑改為腳本所在目錄(Windows 可攜)。

### 資料
- master 549 → 829 列;dict 866 → 1219 條;撞名 0。
- `5` 因 ɫ 回歸 initials;冷按直上字面者現為 `' ( ) + , . / 0`。

## 0.1.3 — V3-b 根治:照官方 rime-ipa 拆除標點管線

**關鍵發現:Rime 官方存在 IPA 方案(rime/rime-ipa),其結構驗證了正解——
符號密集 alphabet 必須拆除整條標點管線。** 使用者觀察「尾端/接標點時 .
被吃掉、候選不受影響」即 punct_segmentor 把 . 搶成無候選死段的症狀。

### 引擎結構(仿官方 rime-ipa,保留本案注音式設計)
- processors 移除 recognizer、punctuator;segmentors 移除 matcher、punct_segmentor;
  translators 移除 punct_translator。標點鍵一律走 speller/碼表。
- 新增 echo_translator:無任何候選時以原始輸入碼為候選——未匹配段(如 t.)
  **可見**且可上屏,不再有隱形狀態。

### initials 機制(官方 rime-ipa 同款)
- 新增 speller/initials:僅列「有真實碼以其起首」的鍵(自動生成),
  自指候選不賦予起始資格;另手動排除 `.`。
- **冷按 ' ( ) + , . / 0 5 直接上屏字面**(不進組字)——句號、逗號打句子零阻力;
  這些字元在碼中間(\t.、\c,、\o/、\'1、\T()照常有效。
- 音節界 `.` 無損:冷按上屏的字面句號即該符號本身;`sbr` 碼仍可達。

### 自指候選調整
- HIGH(字面第一)縮為 initials 內可達者:`. | \ < > -`;
  `, ( ) ' / +` 改由「冷按直上」承接,移除死項。dict 872 → 866。
- LOW(字面第二)不變:`? ~ = : % " @ & { } \` _ !`。

### 待複測(V8)
- `\t.` → 應即時顯示候選 ʈ(核心案例)。
- `t.` → 應顯示 echo 候選「t.」(可見,不再消失);`t.g`、`t.\` 同理可見。
- 冷按 `.` `,` → 直接上屏字面,不開組字。
- `|` 冷按 → 組字,第一候選字面 |;`|13` → ˩˧。
- numpad 冷按 `1` → 組字出 ɨ(V7 續驗)。
